# 02 — 测试报告与监控

## 1 覆盖率报告

### 1.1 Codecov 集成

在 CI 流水线中已配置 `codecov/codecov-action@v4`。需要在 [codecov.io](https://codecov.io) 关联仓库后获取 `CODECOV_TOKEN`，配置到 GitHub Secrets。

**Codecov 配置（可选）：`codecov.yml`**

```yaml
coverage:
  status:
    project:
      default:
        target: 70%            # 整体覆盖率目标
        threshold: 2%          # 允许下降 2% 不报 failure
    patch:
      default:
        target: 80%            # 新增代码覆盖率目标
        threshold: 5%

comment:
  layout: "reach, diff, flags, files"
  behavior: default
  require_changes: false

ignore:
  - "cmd/main.go"             # 入口文件不计入
  - "pkg/config/config.go"    # 纯配置加载，难以测试
  - "**/*_test.go"
```

### 1.2 本地覆盖率查看

```bash
# 生成 HTML 覆盖率报告
go test ./... -coverprofile=coverage.out -covermode=atomic
go tool cover -html=coverage.out -o coverage.html

# 按包查看覆盖率
go tool cover -func=coverage.out | sort -k3 -n

# 只看 Service 层
go test ./service/... -coverprofile=svc_cov.out
go tool cover -func=svc_cov.out
```

## 2 测试结果通知

### 2.1 GitHub Actions 内置通知

- **PR Checks：** 每个 Job 的状态直接显示在 PR 页面的 Checks Tab
- **分支保护规则：** main 分支可配置 `Require status checks to pass before merging`，勾选 `lint`、`unit-test`、`integration-test`

### 2.2 失败通知

```yaml
# 在 ci.yml 中添加 failure notification job
notify-failure:
  name: Notify Failure
  runs-on: ubuntu-latest
  needs: [lint, unit-test, integration-test, build]
  if: failure()
  steps:
    - name: Send failure notification
      uses: slackapi/slack-github-action@v1
      with:
        payload: |
          {
            "text": "❌ CI Pipeline Failed\n
            Repository: ${{ github.repository }}\n
            Branch: ${{ github.ref_name }}\n
            Commit: ${{ github.sha }}\n
            Author: ${{ github.actor }}\n
            Run: ${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}"
          }
      env:
        SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

## 3 测试数据看板

### 3.1 关键指标

| 指标 | 来源 | 更新频率 |
|------|------|---------|
| 单元测试通过率 | GitHub Actions | 每次 Push |
| 集成测试通过率 | GitHub Actions | 每次 PR |
| 代码覆盖率趋势 | Codecov | 每次 Push |
| 测试执行时长 | GitHub Actions 日志 | 每次 Push |
| 竞态检测通过率 | GitHub Actions | 每次 Push |

### 3.2 测试执行时长监控

```bash
# CI 中统计测试耗时分布
go test ./... -v -json | \
  jq -r 'select(.Action=="pass" and .Test!=null) | "\(.Test) \(.Elapsed)"' | \
  sort -k2 -n | tail -10
```

### 3.3 失败趋势追踪

建议在每个 Sprint 结束时手动 Review：

```bash
# 查看最近 10 次 CI 运行状态
gh run list --workflow=ci.yml --limit 10 \
  --json name,status,conclusion,createdAt,headBranch \
  --jq '.[] | "\(.createdAt)  \(.status)  \(.conclusion)  \(.headBranch)"'
```

## 4 测试数据持久化

### 4.1 Artifact 保留策略

| Artifact | 保留天数 | 说明 |
|----------|---------|------|
| coverage-report | 90 天 | 覆盖率数据归档 |
| integration-test-log | 30 天 | 集成测试日志 |
| manage_system_linux | 30 天 | 构建产物 |

### 4.2 测试结果归档脚本

```bash
#!/bin/bash
# scripts/archive_test_results.sh
# 本地归档测试结果（用于离线分析）

ARCHIVE_DIR="test_archive/$(date +%Y-%m-%d_%H-%M-%S)"
mkdir -p "$ARCHIVE_DIR"

# 单元测试结果
go test ./... -v -json > "$ARCHIVE_DIR/unit_test.json" 2>&1
go tool cover -func=coverage.out > "$ARCHIVE_DIR/coverage.txt" 2>&1

# 集成测试结果
go test ./... -tags=integration -v -json > "$ARCHIVE_DIR/integration_test.json" 2>&1

# 性能基准
go test ./... -bench=. -benchmem > "$ARCHIVE_DIR/benchmark.txt" 2>&1

echo "Archived to $ARCHIVE_DIR"
```

## 5 性能基准测试

```go
// dao/borrow_test.go (benchmark)
func BenchmarkBorrowDAO_FindPage(b *testing.B) {
    db := setupBenchDB(b)
    dao := NewBorrowDAO(db)
    b.ResetTimer()
    for i := 0; i < b.N; i++ {
        dao.FindPage(0, 10, 0, 0, "")
    }
}

func BenchmarkBorrowService_Approve(b *testing.B) {
    svc := setupBenchService(b)
    b.ResetTimer()
    for i := 0; i < b.N; i++ {
        svc.Approve(ctx, 1, true, 1, "")
    }
}
```

运行基准测试：

```bash
go test ./... -bench=. -benchmem -count=5 | tee bench.txt

benchstat bench.txt
```
