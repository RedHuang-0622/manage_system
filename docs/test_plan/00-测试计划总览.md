# 00 — 测试计划总览

## 1 测试策略

本项目采用**测试金字塔**模型组织测试，自底向上逐层覆盖：

```
          ┌─────────────┐
          │   E2E / API │  ← 全链路业务场景（少量，覆盖核心闭环）
          ├─────────────┤
          │  集成测试    │  ← 多模块协作 + 真实 DB/Redis（中等数量）
          ├─────────────┤
          │  单元测试    │  ← DAO / Service / pkg 独立测试（最多）
          └─────────────┘
```

**覆盖率目标：**

| 层级 | 目标覆盖率 | 说明 |
|------|-----------|------|
| DAO | ≥ 80% | CRUD、分页、事务、行锁 |
| Service | ≥ 85% | 业务逻辑、边界条件、错误路径 |
| pkg (jwt/errcode/response) | ≥ 90% | 纯函数居多，容易覆盖 |
| Controller | ≥ 70% | 参数校验、响应格式 |

## 2 测试工具

| 工具 | 用途 | 说明 |
|------|------|------|
| `go test` | 测试运行器 | 内置，支持 `-race` 竞态检测 |
| `testify` | 断言 + Mock | `assert` / `require` / `mock` |
| `go-sqlmock` | SQL Mock | 模拟 GORM 操作，不依赖真实 DB |
| `httptest` | HTTP 测试 | Gin 框架内置，模拟 HTTP 请求 |
| `miniredis` | Redis Mock | 内存 Redis，用于缓存测试 |
| `docker-compose` | 集成测试环境 | 启动真实 MySQL + Redis |

## 3 测试环境

### 3.1 单元测试环境

```go
// 无需外部依赖，使用 Mock
func TestUserDAO_Create(t *testing.T) {
    db, mock, _ := sqlmock.New()
    gormDB, _ := gorm.Open(mysql.New(mysql.Config{Conn: db, SkipInitializeWithVersion: true}))
    dao := NewUserDAO(gormDB)
    // ...
}
```

### 3.2 集成测试环境

```yaml
# docker-compose.test.yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: test
      MYSQL_DATABASE: lab_manage_test
    ports: ["3307:3306"]

  redis:
    image: redis:7-alpine
    ports: ["6380:6379"]
```

**测试数据库名：** `lab_manage_test`（与生产隔离）  
**配置分离：** `conf/config.test.yaml`

### 3.3 测试运行命令

```bash
# 单元测试（无需外部依赖）
go test ./dao/... ./service/... ./pkg/... -v -cover

# 竞态检测
go test ./... -race

# 集成测试（需要 docker-compose 环境）
go test ./... -tags=integration -v

# 覆盖率报告
go test ./... -coverprofile=coverage.out
go tool cover -html=coverage.out
```

## 4 测试数据管理

### 4.1 种子数据

每个测试文件使用 `setupTest` 函数准备数据：

```go
func setupTestDB(t *testing.T) *gorm.DB {
    db := connectTestDB()
    db.AutoMigrate(&models.SysUser{}, &models.SysRole{},
        &models.LabEquipment{}, &models.BorrowRecord{})

    // 插入基础种子数据
    db.Create(&models.SysRole{ID: 1, RoleName: "super_admin"})
    db.Create(&models.SysRole{ID: 2, RoleName: "lab_admin"})
    db.Create(&models.SysRole{ID: 3, RoleName: "member"})
    return db
}
```

### 4.2 数据清理

```go
func teardownTestDB(db *gorm.DB) {
    db.Exec("DELETE FROM borrow_records")
    db.Exec("DELETE FROM lab_equipments")
    db.Exec("DELETE FROM sys_users")
    db.Exec("DELETE FROM sys_roles")
}
```

### 4.3 测试用例命名规范

```
Test<模块>_<函数>_<场景>_<期望>

示例:
TestUserDAO_FindByUsername_UserExists_ReturnUser
TestBorrowService_Approve_StockInsufficient_ReturnError
TestAuthMiddleware_NoToken_Return401
```
