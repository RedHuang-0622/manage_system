### 4.1 悲观锁验证

| 编号 | 测试用例 | 初始状态 | 并发操作 | 预期 |

| B-CONC-01 | 3 库存 vs 5 个审批 (qty=1) | avail=3 | 5 goroutine 同时审批 5 个工单 | 3 个成功 (avail=0), 2 个返回库存不足 |
| B-CONC-02 | 3 库存 vs 2 个审批 (qty=2) | avail=3 | 2 goroutine 同时审批 2 个工单 (各 qty=2) | 1 个成功 (avail=1), 1 个返回库存不足 |
| B-CONC-03 | 1 库存 vs 10 个审批 (qty=1) | avail=1 | 10 goroutine 同时审批 | 1 个成功 (avail=0), 9 个返回库存不足 |
| B-CONC-04 | 审批 + 归还并发 | avail=1 | 审批(qty=1) + 归还(qty=1) 同时 | 最终状态一致，avail 正确 |
| B-CONC-05 | 死锁验证 — 交叉锁定顺序 | — | 事务1: lock borrow#1→equip#A; 事务2: lock borrow#2→equip#A | 统一先 borrow 后 equip 顺序 → 不死锁 |
| B-CONC-06 | 库存最终一致性 | avail=10, 100 并发各 qty=1 | — | sum(成功) ≤ 10, 最终 avail = 10 - sum(成功), 无超卖 |

### 4.2 并发测试实现

```go
func TestBorrowService_Approve_Concurrent_NoOversell(t *testing.T) {
    // 准备：1 个设备，available_stock=3，5 个申请中工单
    equip := createEquipment(t, db, 3)
    records := make([]*models.BorrowRecord, 5)
    for i := 0; i < 5; i++ {
        records[i] = createBorrowRecord(t, db, equip.ID, 1, "申请中")
    }

    // 并发审批
    var wg sync.WaitGroup
    results := make(chan error, 5)
    for i := 0; i < 5; i++ {
        wg.Add(1)
        go func(idx int) {
            defer wg.Done()
            err := service.Approve(ctx, records[idx].ID, true, 1, "")
            results <- err
        }(i)
    }
    wg.Wait()
    close(results)

    // 统计
    successCount := 0
    for err := range results {
        if err == nil {
            successCount++
        } else {
            require.Equal(t, ErrStockInsufficient, err)
        }
    }
    require.Equal(t, 3, successCount)

    // 验证最终库存
    updatedEquip, _ := equipDAO.FindByID(equip.ID)
    require.Equal(t, uint(0), updatedEquip.AvailableStock)
}
```
