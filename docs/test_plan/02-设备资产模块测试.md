# 02 — 设备资产模块测试

## 1 测试范围

| 层级 | 测试内容 |
|------|---------|
| DAO | 设备 CRUD、分页查询、条件筛选、FOR UPDATE 行锁 |
| Service | 库存管理逻辑、缓存读写编排、缓存失效触发 |
| Controller | 参数校验、响应格式、权限检查 |

## 3 Service 层测试

|------|---------|------|----------|------|
| E-SVC-01 | Create — 正常 | name="GPU", total=10 | DAO Insert 成功 | available = total = 10，status=1，返回设备 |
| E-SVC-02 | Create — 触发缓存失效 | — | — | ListPage 缓存被清除 |

| E-SVC-11 | Disable — 正常 | id=1 | 无未归还借阅 | status=0，缓存失效 |
| E-SVC-12 | Disable — 有未归还借阅 | id=1 | borrowDAO.Count 返回 >0 | 返回错误 |
| E-SVC-13 | 缓存失效 — 更新后详情缓存 | Update(id=1) | — | equip:detail:1 缓存被删除 |
| E-SVC-14 | 缓存失效 — 创建后列表缓存 | Create(...) | — | equip:list:* 缓存被批量清除 |

### 3.2 缓存读写

| 编号 | 测试用例 | 输入 | Mock 行为 | 预期 |
|------|---------|------|----------|------|
| E-CACHE-01 | ListPage — 读缓存命中 | page=1, size=12 | Redis Get 返回 JSON | 返回反序列化的 PageResult，不查 DB |
| E-CACHE-02 | ListPage — 读缓存未命中 | page=1, size=12 | Redis Get 返回 nil | 查 DB 后 Set 到 Redis |
| E-CACHE-03 | ListPage — 缓存 JSON 解析失败 | — | Redis Get 返回非法 JSON | 查 DB 覆盖缓存，返回正确数据 |
| E-CACHE-04 | ListPage — Redis 连接失败 | — | Redis Get 返回连接错误 | 降级查 DB，返回正确数据（不断路） |
| E-CACHE-05 | GetByID — 缓存命中 | id=1 | Redis Get 命中 | 返回缓存数据 |
| E-CACHE-06 | GetByID — 缓存未命中 | id=1 | Redis Get 未命中 | 查 DB → Set 缓存 → 返回 |
| E-CACHE-07 | 缓存 Key 格式正确性 | page=2, size=12, kw="GPU", cat="服务器", status=1, only=0 | — | Key = "equip:list:2:12:<hash_kw>:服务器:1:0" |
| E-CACHE-08 | 写操作异步失效 — 不阻塞主流程 | Create 触发 invalidateListCache | Redis 操作在 goroutine 中 | Create 立即返回，不等缓存删除 |
