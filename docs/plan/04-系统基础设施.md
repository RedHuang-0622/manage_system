# 04 — 系统基础设施

## 1 模块定位

系统基础设施层（Shared Kernel / Cross-cutting Concerns）封装所有业务模块共同依赖的通用组件。它不承载业务逻辑，而是为业务模块提供日志、配置、响应格式、错误码、中间件等横切能力。

在微服务架构中，这一层通常以**公共 Library** 的形式被所有服务引用，或下沉到 **API Gateway / Service Mesh**。

**涉及的包/文件：**
- `pkg/config/config.go` — Viper 配置加载
- `pkg/zap/zap.go` — Zap 日志初始化 + Lumberjack 切割
- `pkg/response/response.go` — 统一 JSON 响应格式
- `pkg/errcode/errcode.go` — 错误码常量与消息映射
- `pkg/redis/redis.go` — Redis 连接池
- `pkg/jwt/jwt.go` — JWT 工具（被 IAM 模块使用）
- `router/middleware/recovery.go` — 全局 Panic 捕获
- `router/middleware/logger.go` — 请求日志记录
- `conf/config.yaml` — 全局配置文件

## 2 日志系统（Zap + Lumberjack）

### 2.1 初始化

```go
func InitLogger(cfg LogConfig) *zap.Logger {
    level := zap.NewAtomicLevel()
    level.UnmarshalText([]byte(cfg.Level))

    encoderConfig := zapcore.EncoderConfig{
        TimeKey:        "ts",
        LevelKey:       "level",
        NameKey:        "logger",
        CallerKey:      "caller",
        MessageKey:     "msg",
        StacktraceKey:  "stacktrace",
        LineEnding:     zapcore.DefaultLineEnding,
        EncodeLevel:    zapcore.LowercaseLevelEncoder,
        EncodeTime:     zapcore.ISO8601TimeEncoder,
        EncodeDuration: zapcore.SecondsDurationEncoder,
        EncodeCaller:   zapcore.ShortCallerEncoder,
    }

    // Lumberjack 日志切割
    writer := &lumberjack.Logger{
        Filename:   cfg.Path,       // "logs/app.log"
        MaxSize:    cfg.MaxSize,    // 100 MB
        MaxBackups: cfg.MaxBackups, // 30
        MaxAge:     cfg.MaxAge,     // 7 天
        Compress:   true,
        LocalTime:  true,
    }

    core := zapcore.NewCore(
        zapcore.NewJSONEncoder(encoderConfig),
        zapcore.NewMultiWriteSyncer(zapcore.AddSync(os.Stdout), zapcore.AddSync(writer)),
        level,
    )

    return zap.New(core, zap.AddCaller(), zap.AddStacktrace(zapcore.ErrorLevel))
}
```

### 2.2 日志等级使用规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| Debug | 开发调试 | SQL 语句、变量值 |
| Info | 正常业务流程 | 请求/响应、业务关键操作（登录、审批、归还） |
| Warn | 可恢复异常 | 权限拒绝、参数校验失败、缓存未命中 |
| Error | 需要关注的错误 | 数据库连接失败、事务回滚、库存不足 |
| Fatal | 致命错误 | 配置加载失败、端口占用 |

### 2.4 请求日志中间件

```go
func Logger(logger *zap.Logger) gin.HandlerFunc {
    return func(c *gin.Context) {
        start := time.Now()
        c.Next()
        status := c.Writer.Status()
        fields := []zap.Field{
            zap.String("method", c.Request.Method),
            zap.String("path", c.Request.URL.Path),
            zap.Int("status", status),
            zap.Duration("latency_ms", time.Since(start)),
            zap.String("ip", c.ClientIP()),
        }
        if userID, ok := c.Get("user_id"); ok {
            fields = append(fields, zap.Any("user_id", userID))
        }

        // 根据 HTTP 状态码选择日志级别
        switch {
        case status >= 500:
            logger.Error("request", fields...)
        case status >= 400:
            logger.Warn("request", fields...)
        default:
            logger.Info("request", fields...)
        }
    }
}
```

## 3 全局异常恢复中间件（Recovery）

**位置：** `router/middleware/recovery.go`

```go
func Recovery(logger *zap.Logger) gin.HandlerFunc {
    return func(c *gin.Context) {
        defer func() {
            if err := recover(); err != nil {
                logger.Error("panic_recovered",
                    zap.Any("error", err),
                    zap.String("stack", string(debug.Stack())),
                    zap.String("path", c.Request.URL.Path),
                    zap.String("method", c.Request.Method),
                    zap.String("ip", c.ClientIP()),
                )
                c.AbortWithStatusJSON(500, gin.H{
                    "code": errcode.ErrInternalServer, "msg": errcode.GetMsg(errcode.ErrInternalServer), "data": nil,
                })
            }
        }()
        c.Next()
    }
}
```

**设计要点：**
- 捕获所有未预期 panic，防止进程崩溃
- 记录完整堆栈便于排查
- 前端只看到通用错误消息，不暴露内部细节

## 4 统一响应格式

**文件：** `pkg/response/response.go`

### 4.1 基础结构

```go
type Response struct {
    Code int         `json:"code"`
    Msg  string      `json:"msg"`
    Data interface{} `json:"data"`
}
```

### 4.2 常用方法

```go
func Success(c *gin.Context, data interface{}) {
    c.JSON(200, Response{Code: 0, Msg: "success", Data: data})
}

func Created(c *gin.Context, data interface{}) {
    c.JSON(201, Response{Code: 0, Msg: "创建成功", Data: data})
}

func Error(c *gin.Context, httpStatus int, code int, msg string) {
    c.JSON(httpStatus, Response{Code: code, Msg: msg, Data: nil})
}

func PageResult(c *gin.Context, total int64, page, pageSize int, list interface{}) {
    c.JSON(200, Response{Code: 0, Msg: "success", Data: gin.H{
        "total": total, "page": page, "page_size": pageSize, "list": list,
    }})
}
```

### 4.3 通用分页结构体

三个业务模块共用统一的分页请求/响应基类，避免各自定义冲突：

```go
// pkg/response/response.go

// PageRequest 为所有分页查询接口的 Query 参数基类
type PageRequest struct {
    Page     int `form:"page"      json:"page"      binding:"omitempty,gte=1"`
    PageSize int `form:"page_size" json:"page_size" binding:"omitempty,gte=1,lte=100"`
}

// PageResult 为分页查询的统一响应体
type PageResult struct {
    Total    int64       `json:"total"`
    Page     int         `json:"page"`
    PageSize int         `json:"page_size"`
    List     interface{} `json:"list"`
}

// Normalize 设置分页默认值并计算 offset
func (p *PageRequest) Normalize() (offset, limit int) {
    if p.Page <= 0 { p.Page = 1 }
    if p.PageSize <= 0 { p.PageSize = 10 }
    return (p.Page - 1) * p.PageSize, p.PageSize
}
```

## 5 错误码体系

### 5.1 分段规则

| 范围 | 类别 | 说明 |
|------|------|------|
| 0 | 成功 | 正常响应 |
| 1001 - 1999 | 通用错误 | 参数校验、资源不存在、冲突 |

### 5.4 Validator 错误翻译

```go
func translateValidationError(err error) string {
    if validationErrors, ok := err.(validator.ValidationErrors); ok {
        var msgs []string
        for _, e := range validationErrors {
            msgs = append(msgs, fmt.Sprintf("%s: 校验规则 %s 不通过", e.Field(), e.Tag()))
        }
        return strings.Join(msgs, "; ")
    }
    return "参数格式错误"
}

type Config struct {
    Server ServerConfig   `mapstructure:"server"`
    MySQL  MySQLConfig    `mapstructure:"mysql"`
    Redis  RedisConfig    `mapstructure:"redis"`
    JWT    JWTConfig      `mapstructure:"jwt"`
    Casbin CasbinConfig   `mapstructure:"casbin"`

}

type ServerConfig struct { Port int; Mode string }
type MySQLConfig struct {
    Host, User, Password, Database, Charset string
    Port, MaxIdleConns, MaxOpenConns, ConnMaxLifetime int
}
type RedisConfig struct { Addr, Password string; DB, PoolSize, MinIdleConns int }

type LogConfig struct { Path, Level string; MaxSize, MaxBackups, MaxAge int }
```

### 6.3 Viper 加载

```go
func Load(configPath string) *Config {
    v := viper.New()
    v.SetConfigFile(configPath)
    v.SetConfigType("yaml")
    v.SetEnvPrefix("LAB")
    v.SetEnvKeyReplacer(strings.NewReplacer(".", "_"))
    v.AutomaticEnv()
    v.ReadInConfig()

    var cfg Config
    v.Unmarshal(&cfg)
    setDefaults(&cfg)
    return &cfg
}
```

### 6.3.1 配置文件路径约束

`config.Load("conf/config.yaml")` 使用**相对路径**，要求运行时工作目录为项目根目录。

**各环境启动方式：**

| 环境 | 启动命令 | 工作目录 |
|------|---------|---------|
| 本地开发 | `go run cmd/main.go` | 项目根目录 `manage_system/` |
| 本地编译 | `go build -o server cmd/main.go && ./server` | 项目根目录 |
| Docker | 见下方 Dockerfile 示例 | `/app`（需 COPY conf/ 到镜像） |

**Dockerfile 示例：**
```dockerfile
FROM golang:1.21-alpine AS builder
WORKDIR /build
COPY . .
RUN go build -o server cmd/main.go

FROM alpine:3.19
WORKDIR /app
COPY --from=builder /build/server .
COPY --from=builder /build/conf/ ./conf/    # ← 关键：conf 目录必须在工作目录下
EXPOSE 8080
CMD ["./server"]
```

> **未来改进（V2.0）：** 通过环境变量 `LAB_CONFIG_PATH` 指定绝对路径，消除对工作目录的依赖。

### 6.4 环境变量覆盖

Viper 的 `AutomaticEnv()` 允许环境变量覆盖 config.yaml 中的值。前缀为 `LAB_`，句点替换为下划线（如 `mysql.password` → `LAB_MYSQL_PASSWORD`）。

#### 环境变量分级

| 级别 | 变量 | 说明 |
|------|------|------|
| 🔴 **必须覆盖** | `LAB_MYSQL_PASSWORD` | config.yaml 中为空，不覆盖则无法连接数据库 |
| 🔴 **必须覆盖** | `LAB_JWT_SECRET` | config.yaml 中为示例值，生产泄漏将导致 Token 可伪造 |
| 🟡 **强烈建议** | `LAB_REDIS_PASSWORD` | 若 Redis 有密码保护 |
| 🟢 **可选** | `LAB_SERVER_PORT` | 默认 8080，仅在端口冲突时需要 |
| 🟢 **可选** | `LAB_REDIS_ADDR` | 默认 127.0.0.1:6379，仅在非本地 Redis 时需要 |
| 🟢 **可选** | `LAB_LOG_LEVEL` | 默认 info，调试时可设为 debug |

#### 开发环境
```bash
export LAB_MYSQL_PASSWORD=dev_pass
export LAB_JWT_SECRET=dev-secret-key-at-least-32-chars-long
```

#### 生产环境（示例）
```bash
export LAB_SERVER_PORT=9090
export LAB_MYSQL_PASSWORD=prod_secure_pass
export LAB_JWT_SECRET=$(openssl rand -base64 32)  # 随机生成 32+ 字符密钥
export LAB_REDIS_ADDR=redis-cluster:6379
export LAB_REDIS_PASSWORD=prod_redis_pass
export LAB_LOG_LEVEL=info
```

## 7 Redis 连接池

```go
func InitRedis(cfg RedisConfig) *redis.Client {
    rdb := redis.NewClient(&redis.Options{
        Addr: cfg.Addr, Password: cfg.Password, DB: cfg.DB,
        PoolSize: cfg.PoolSize, MinIdleConns: cfg.MinIdleConns,
        MaxRetries: 3, DialTimeout: 5*time.Second,
        ReadTimeout: 3*time.Second, WriteTimeout: 3*time.Second,
    })
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
    if err := rdb.Ping(ctx).Err(); err != nil {
        panic("failed to connect redis: " + err.Error())
    }
    return rdb
}
```

## 8 中间件链注册顺序

在 `router/router.go` 中的注册顺序决定请求处理链：

```
HTTP 请求
  → Recovery        (① 最外层：捕获所有 Panic)
    → Logger        (② 记录请求耗时)
      → [公开路由组]
          /health         (无中间件)
          /auth/login     (无中间件)
          /auth/logout    (Auth → Casbin)
      → [受保护路由组]
          → Auth        (③ JWT 认证 → 注入 user_id/role_name)
            → Casbin    (④ 权限鉴权 → Enforce)
              → Controller
```
