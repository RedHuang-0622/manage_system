# 03 — 模块间内部接口

## 1 设计说明

当前项目为**单体架构**，模块间通信通过同进程内的方法调用完成。本文档定义 DAO 层和 Service 层的接口契约——这些接口在单体中是 Go `interface`，未来拆分微服务时可作为 gRPC Proto 的定义依据。

**当前调用方式 vs 未来演进：**

| 层级 | 当前（单体） | 未来（微服务） |
|------|------------|---------------|
| DAO → DB | `gorm.DB` 直接操作 | 各服务独立 DB 连接 |
| Service → DAO | 同进程 interface 调用 | 同服务内，不变 |
| Service → Service (跨模块) | 直接 import 调用 | gRPC / 消息队列 |
| Borrow → Equipment DAO (事务) | 同一 `*gorm.DB` 传入 | Saga / Outbox 模式 |

## 2 DAO 层接口契约

### 2.1 UserDAO

```go
// dao/user.go
type UserDAO interface {
    // 创建用户
    Create(user *models.SysUser) error

    // 按 ID 查询（默认 Preload Role）
    FindByID(id uint) (*models.SysUser, error)

    // 按用户名查询（用于登录）
    FindByUsername(username string) (*models.SysUser, error)

    // 分页查询
    // offset: 偏移量, limit: 每页条数
    // keyword: 模糊匹配 username 或 real_name
    // status: -1=全部, 0=禁用, 1=启用
    // roleID: 0=全部
    FindPage(offset, limit int, keyword string, status int, roleID uint) ([]models.SysUser, int64, error)

    // 部分字段更新
    UpdateFields(id uint, updates map[string]interface{}) error
}
```

### 2.2 RoleDAO

```go
// dao/role.go
type RoleDAO interface {
    FindAll() ([]models.SysRole, error)
```

### 2.3 EquipmentDAO

```go
// dao/equipment.go
type EquipmentDAO interface {
    // ── 基本 CRUD ──
    Insert(equip *models.LabEquipment) error
    FindByID(id uint) (*models.LabEquipment, error)
    FindPage(offset, limit int, keyword, category string, status int, onlyAvailable bool) ([]models.LabEquipment, int64, error)
    Update(equip *models.LabEquipment) error
    UpdateFields(id uint, updates map[string]interface{}) error

    // ── 事务内操作（供 BorrowService 跨模块调用）──
    // 行级锁查询：SELECT ... FOR UPDATE
    GetForUpdate(tx *gorm.DB, id uint) (*models.LabEquipment, error)
    // 事务内更新（tx 为同一个 GORM 事务）
    UpdateWithTx(tx *gorm.DB, equip *models.LabEquipment) error
}
```

**跨模块调用说明：**

```
BorrowService.Approve()
    │
    ├── borrowDAO.GetForUpdate(tx, borrowID)     ← 同一 tx
    ├── equipDAO.GetForUpdate(tx, equipID)       ← 同一 tx，跨 DAO
    ├── equipDAO.UpdateWithTx(tx, equip)         ← 同一 tx，跨 DAO
    └── borrowDAO.UpdateWithTx(tx, record)       ← 同一 tx

全部在 db.Transaction(func(tx *gorm.DB) error { ... }) 内执行
两个 DAO 共享同一个 *gorm.DB 事务实例
```

### 2.4 BorrowDAO

```go
// dao/borrow.go
type BorrowDAO interface {
    // ── 基本 CRUD ──
    Insert(record *models.BorrowRecord) error
    FindByID(id uint) (*models.BorrowRecord, error)
    FindPage(offset, limit int, userID, equipID uint, status string) ([]models.BorrowRecord, int64, error)

    // ── 业务查询 ──
    // 用户在指定设备上的活跃借阅数（申请中 + 已借出）
    CountActiveByUserAndEquipment(userID, equipID uint) (int64, error)
    // 指定设备在指定状态的借阅数（如下架前检查"已借出"）
    CountByEquipmentAndStatus(equipID uint, status string) (int64, error)

    // ── 事务内操作 ──
    GetForUpdate(tx *gorm.DB, id uint) (*models.BorrowRecord, error)    // 行锁
    UpdateWithTx(tx *gorm.DB, record *models.BorrowRecord) error
}
```

## 4 跨模块依赖关系

```
IAMService
  ├── 依赖: UserDAO, RoleDAO, jwt.Service, redis.Client
  └── 被依赖: AuthMiddleware (通过 c.Get("claims"))

EquipmentService
  ├── 依赖: EquipmentDAO, redis.Client
  ├── 依赖: BorrowDAO (Disable 时检查未归还借阅)
  └── 被依赖: BorrowService (通过 EquipmentDAO.GetForUpdate / UpdateWithTx)

BorrowService
  ├── 依赖: BorrowDAO, EquipmentDAO, redis.Client
  ├── 调用 EquipmentDAO.GetForUpdate(tx, id)    ← DB 事务内跨 DAO
  └── 调用 EquipmentDAO.UpdateWithTx(tx, equip) ← DB 事务内跨 DAO
```

### 4.1 跨模块调用矩阵

| 调用方 | 被调用方 | 方式 | 是否事务性 |
|--------|---------|------|-----------|
| BorrowService | EquipmentDAO.GetForUpdate | 直接方法调用（注入同一 *gorm.DB tx） | ✅ 同一事务 |
| BorrowService | EquipmentDAO.UpdateWithTx | 直接方法调用（注入同一 *gorm.DB tx） | ✅ 同一事务 |
| EquipmentService | BorrowDAO.CountByEquipmentAndStatus | 直接方法调用（独立查询） | ❌ |
| AuthMiddleware | IAMService (via jwt) | 读取 Context 中 JWT Claims | ❌ |
| CasbinMiddleware | Casbin Enforcer | 读取 Context 中 role_name | ❌ |

## 5 微服务拆分时的接口演进路径

如果将来从单体拆分为微服务，以下 DAO 跨模块调用需要改为远程调用：

### 5.1 Borrow → Equipment（当前在同一 DB 事务）

**当前（单体）：**
```go
s.db.Transaction(func(tx *gorm.DB) error {
    equip, _ := equipDAO.GetForUpdate(tx, equipID)
    equip.AvailableStock -= quantity
    equipDAO.UpdateWithTx(tx, equip)    // 同一个 MySQL 连接
    borrowDAO.UpdateWithTx(tx, record)  // 同一个 MySQL 连接
})
```

**未来（微服务 — Saga 模式）：**
```protobuf
// equipment.proto
service EquipmentService {
    rpc ReserveStock(ReserveStockReq) returns (ReserveStockResp);  // 扣库存
    rpc ReleaseStock(ReleaseStockReq) returns (ReleaseStockResp);  // 补偿：恢复库存
}

// borrow.proto
service BorrowService {
    rpc ApproveBorrow(ApproveBorrowReq) returns (ApproveBorrowResp);
}
```

**Saga 编排：**
```
1. BorrowService.ApproveBorrow()
     → call EquipmentService.ReserveStock()  // 正向操作
     → update borrow_records status
     → if EquipmentService fails: compensating action
     → call EquipmentService.ReleaseStock()   // 补偿操作
```

### 5.2 数据所有权拆分

| 当前表 | 当前归属 | 微服务拆分后归属 |
|--------|---------|-----------------|
| sys_users | IAM（整表） | Auth Service (username, password_hash, status) + User Service (real_name, email, phone, role_id) |
| sys_roles | IAM | Auth Service |
| casbin_rule | IAM | Auth Service |
| lab_equipments | Equipment | Equipment Service（独立 DB） |
| borrow_records | Borrow | Borrow Service（独立 DB） |

### 5.3 接口定义演进对照

| 当前（Go interface） | 未来（gRPC / REST） |
|---------------------|---------------------|
| `UserDAO.FindByUsername(name)` | `GET /internal/users?username=xxx` 或 `rpc GetUserByUsername` |
| `EquipmentDAO.GetForUpdate(tx, id)` | `rpc ReserveStock(id, qty)` (Saga 正向操作) |
| `EquipmentDAO.UpdateWithTx(tx, equip)` | 含在 ReserveStock / ReleaseStock 内部 |
| `BorrowDAO.CountByEquipmentAndStatus(id, status)` | `rpc GetEquipmentBorrowCount(id, status)` |
